## Documentación Oficial

### Python

**Python Official Docs**
- URL: https://docs.python.org/3/
- Descripción: Documentación oficial completa de Python
- Nivel: Todos los niveles
- Qué consultar:
  - Tutorial oficial (para principiantes)
  - Library Reference (para todas las bibliotecas estándar)
  - Language Reference (para detalles del lenguaje)

**Python Enhancement Proposals (PEPs)**
- URL: https://peps.python.org/
- Descripción: Propuestas que definen el lenguaje y sus estándares
- PEPs esenciales:
  - PEP 8: Style Guide for Python Code
  - PEP 20: The Zen of Python
  - PEP 257: Docstring Conventions
  - PEP 484: Type Hints
  - PEP 526: Syntax for Variable Annotations

### Bibliotecas Core

**Pandas Documentation**
- URL: https://pandas.pydata.org/docs/
- Descripción: Documentación oficial de pandas
- Recursos clave:
  - Getting Started tutorials
  - User Guide (por tema)
  - API Reference (completa)
  - 10 minutes to pandas (tutorial rápido)

**NumPy Documentation**
- URL: https://numpy.org/doc/stable/
- Descripción: Documentación oficial de NumPy
- Recursos clave:
  - NumPy for absolute beginners
  - NumPy fundamentals

**PyArrow Documentation**
- URL: https://arrow.apache.org/docs/python/
- Descripción: Python bindings para Apache Arrow
- Temas importantes:

**SQLAlchemy Documentation**

## Libros Recomendados

### Fundamentos de Python

**"Python Crash Course" - Eric Matthes**
- Nivel: Principiante
- Páginas: 544
- Descripción: Introducción práctica a Python con proyectos reales
- Por qué leerlo: Mejor libro para comenzar desde cero
- Temas: Sintaxis básica, estructuras de datos, OOP, archivos, testing

**"Fluent Python" - Luciano Ramalho**
- Nivel: Intermedio a Avanzado
- Páginas: 792
- Descripción: Python idiomático y características avanzadas
- Por qué leerlo: Para escribir código pythonic profesional
- Temas: Data structures, functions, OOP, metaprogramming, concurrency

**"Effective Python: 90 Specific Ways to Write Better Python" - Brett Slatkin**
- Nivel: Intermedio
- Páginas: 400
- Descripción: Best practices y patterns de Python
- Por qué leerlo: Mejora calidad del código inmediatamente
- Temas: Pythonic thinking, functions, classes, comprehensions, testing

### Data Engineering

**"Python for Data Analysis" - Wes McKinney**
- Nivel: Principiante a Intermedio
- Páginas: 550
- Descripción: Libro del creador de pandas
- Por qué leerlo: Referencia definitiva para análisis de datos con Python
- Temas: NumPy, pandas, data cleaning, transformation, visualization
- Nota: 3ra edición (2022) actualizada para pandas 2.0

**"Data Engineering with Python" - Paul Crickard**
- Nivel: Intermedio
- Páginas: 400
- Descripción: Guía completa de data engineering con Python
- Por qué leerlo: Cubre todo el ciclo de vida de data engineering
- Temas: ETL, data lakes, streaming, orchestration, testing

**"Designing Data-Intensive Applications" - Martin Kleppmann**
- Nivel: Intermedio a Avanzado
- Páginas: 616
- Descripción: Fundamentos de sistemas de datos distribuidos
- Por qué leerlo: Entender arquitecturas de datos modernas
- Temas: Replication, partitioning, transactions, batch/stream processing
- Nota: No es específico de Python pero esencial para arquitecturas

**"The Data Engineering Handbook" - DataEngineer.io**
- Nivel: Todos los niveles
- Formato: Online/GitHub
- URL: https://github.com/DataEngineer-io/data-engineer-handbook
- Descripción: Handbook open-source comunitario
- Por qué consultarlo: Actualizado constantemente, ejemplos prácticos

### Advanced Topics

**"High Performance Python" - Micha Gorelick, Ian Ozsvald**
- Nivel: Avanzado
- Páginas: 468
- Descripción: Optimización y performance de código Python
- Por qué leerlo: Para procesar grandes volúmenes de datos eficientemente
- Temas: Profiling, Cython, NumPy optimization, multiprocessing, distributed computing

**"Architecture Patterns with Python" - Harry Percival, Bob Gregory**
- Nivel: Intermedio a Avanzado
- Páginas: 300
- Descripción: Patterns arquitectónicos y DDD con Python
- Por qué leerlo: Para diseñar sistemas mantenibles y testables
- Temas: Repository pattern, Unit of Work, Event-driven architecture

## Tips para Maximizar el Aprendizaje

### Estrategias Efectivas

1. **Aprender Haciendo**
   - No solo leer, escribir código todos los días
   - "Tutorial hell" es real: construye proyectos propios
   - Regla 70/30: 70% coding, 30% teoría

2. **Proyectos Propios**
   - Construir mini-pipeline con tus propios datos
   - Automatizar algo tedioso de tu trabajo
   - Contribuir a proyectos open source

3. **Documentar Aprendizaje**
   - Blog personal con TILs (Today I Learned)
   - GitHub con proyectos comentados
   - Enseñar a otros (mejor manera de aprender)

4. **Consistencia sobre Intensidad**
   - 1 hora diaria > 7 horas un solo día
   - Usar técnica Pomodoro (25 min focus, 5 min break)
   - Establecer rutina diaria

5. **Comunidad**
   - Unirse a Discord/Slack de Python
   - Participar en meetups locales
   - Hacer preguntas en Stack Overflow
   - Twitter/LinkedIn para seguir expertos

6. **Feedback Loop**
   - Code reviews (pedir o hacer)
   - Comparar tu código con expertos
   - Refactorizar código antiguo propio
